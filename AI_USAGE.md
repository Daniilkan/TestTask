# Использование AI в TestTask

В проекте **TestTask** используется искусственный интеллект для классификации товаров на три категории: Эконом, Средние и Премиум. Классификация выполняется с помощью LLM (Large Language Model).

## Как работает классификация

1. **Сбор данных**:
   - Товары парсятся с маркетплейса и сохраняются в файл `products.json`.

2. **Формирование запросов**:
   - Список товаров разбивается на группы, и для каждой группы формируется текстовый запрос.

3. **Запрос к LLM**:
   - Запрос отправляется к API LLM (например, OpenAI), который возвращает категории для каждого товара.

4. **Обновление данных**:
   - Категории товаров обновляются в файле `products.json`.

## Переменные окружения для LLM

Для работы с LLM необходимо настроить следующие переменные в файле `.env`:

- `OPENAI_API_KEY` — API-ключ для доступа к LLM.
- `MOCK_LLM` — использовать ли мок-ответы (1 — да, 0 — нет).

Пример настройки:
```dotenv
OPENAI_API_KEY='ваш_api_ключ'
MOCK_LLM=1
```

### Использование мок-ответов
Если включен режим MOCK_LLM, вместо реального запроса к LLM будут возвращаться заранее подготовленные данные. Это полезно для тестирования и разработки.

### Ограничения
- `Распределение категорий` LLM старается равномерно распределить товары по категориям, но при малом количестве товаров возможны отклонения.
- `Скорость работы` При большом количестве товаров время обработки может увеличиваться.

### Примечания
Убедитесь, что ваш API-ключ активен и имеет достаточные лимиты.
Для реального использования рекомендуется отключить режим MOCK_LLM.
